# 多智能体协同运营自动化系统 (Multi-Agent Collaborative System)

## 项目简介

本项目提供了一个基于主从架构（Supervisor-Worker Architecture）的多智能体协同运营系统原生 Python 实现。系统旨在通过多角色的 AI 智能体协同工作，处理链路较长、逻辑复杂的自动化任务。当前核心应用场景为**自动化行业调研与结构化报告生成**。

通过引入标准化的消息传递协议与独立的角色节点，本系统将传统的黑盒式单体大模型（Single Agent）交互，转化为透明、可审计、高容错的分布式工作流。

---

## 核心解决痛点

1. **突破单体模型瓶颈**：通过角色解耦（如规划、检索、整合），规避单一模型在处理长文本或复杂任务时的上下文遗忘与能力偏移。
2. **产能释放与标准化交付**：将“需求理解 - 数据采集 - 分析过滤 - 结构化排版”的全生命周期自动化，大幅降低人工干预成本，确保最终交付物的高度标准化与客观性。
3. **增强流程容错率与溯源性**：引入消息总线与节点控制，实现步骤间的状态隔离。可精准监控每一子任务的输入与输出，使长链任务具备可追溯性。

---

## 核心架构与逻辑流

系统底层采用 **“规划-执行-整合” (Plan-Execute-Synthesize)** 的宏观工作流：

* **规划层 (Planner Agent)**：
  * **职责**：接收宏观业务指令，执行长链推理（Long-chain Reasoning）。
  * **机制**：运用思维链将主任务进行降维与反向工程，生成结构化的子任务清单（JSON 格式），并明确各节点目标与边界。
* **执行层 (Researcher/Worker Agent)**：
  * **职责**：接收具体子任务，执行局部环境交互与数据处理。
  * **机制**：进入局部的推理与行动循环（ReAct），可挂载外部工具（搜索引擎、数据库 API）进行信息检索，完成过滤后将结论回传至系统消息总线。
* **整合层 (Synthesizer Agent)**：
  * **职责**：汇总前置离散成果，执行全局校验与成果渲染。
  * **机制**：监听子任务状态，在全量完成后提取消息总线数据，进行交叉验证与信息降噪，最终输出符合业务规范的结构化报告。

## 快速开始

### 运行环境
* Python 3.8 或更高版本
* 本地运行无需复杂的第三方库（底层逻辑为原生实现），实际部署时需接入对应的大型语言模型 API（如 OpenAI API, Gemini API 等）。

### 执行步骤

1. **克隆项目或保存源码**
   将提供的多智能体协同核心代码保存为 `main.py`。

2. **配置大模型接口（可选但建议）**
   在 `BaseAgent._call_llm` 方法中，将模拟返回的逻辑替换为您实际使用的 LLM API 调用逻辑。

3. **运行系统**
   在终端中执行以下命令启动系统：
   ```bash
   python main.py
